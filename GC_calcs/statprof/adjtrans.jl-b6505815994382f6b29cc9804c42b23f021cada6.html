11 """
12 Adjoint
13
14 Lazy wrapper type for an adjoint view of the underlying linear algebra object,
15 usually an `AbstractVector`/`AbstractMatrix`, but also some `Factorization`, for instance.
16 Usually, the `Adjoint` constructor should not be called directly, use [`adjoint`](@ref)
17 instead. To materialize the view use [`copy`](@ref).
18
19 This type is intended for linear algebra usage - for general data manipulation see
20 [`permutedims`](@ref Base.permutedims).
21
22 # Examples
23 ```jldoctest
24 julia> A = [3+2im 9+2im; 0 0]
25 2×2 Matrix{Complex{Int64}}:
26 3+2im 9+2im
27 0+0im 0+0im
28
29 julia> Adjoint(A)
30 2×2 adjoint(::Matrix{Complex{Int64}}) with eltype Complex{Int64}:
31 3-2im 0+0im
32 9-2im 0+0im
33 ```
34 """

77
78 Lazy adjoint (conjugate transposition). Note that `adjoint` is applied recursively to
79 elements.
80
81 For number types, `adjoint` returns the complex conjugate, and therefore it is equivalent to
82 the identity function for real numbers.
83
84 This operation is intended for linear algebra usage - for general data manipulation see
85 [`permutedims`](@ref Base.permutedims).
86
87 # Examples
88 ```jldoctest
89 julia> A = [3+2im 9+2im; 0 0]
90 2×2 Matrix{Complex{Int64}}:
91 3+2im 9+2im
92 0+0im 0+0im
93
94 julia> B = A' # equivalently adjoint(A)
95 2×2 adjoint(::Matrix{Complex{Int64}}) with eltype Complex{Int64}:
96 3-2im 0+0im
97 9-2im 0+0im
98
99 julia> B isa Adjoint
100 true
101
102 julia> adjoint(B) === A # the adjoint of an adjoint unwraps the parent
103 true
104
105 julia> Adjoint(B) # however, the constructor always wraps its argument
106 2×2 adjoint(adjoint(::Matrix{Complex{Int64}})) with eltype Complex{Int64}:
107 3+2im 9+2im
108 0+0im 0+0im
109
110 julia> B[1,2] = 4 + 5im; # modifying B will modify A automatically
111
112 julia> A
113 2×2 Matrix{Complex{Int64}}:
114 3+2im 9+2im
115 4-5im 0+0im
116 ```
117
118 For real matrices, the `adjoint` operation is equivalent to a `transpose`.
119
120 ```jldoctest
121 julia> A = reshape([x for x in 1:4], 2, 2)
122 2×2 Matrix{Int64}:
123 1 3
124 2 4
125
126 julia> A'
127 2×2 adjoint(::Matrix{Int64}) with eltype Int64:
128 1 2
129 3 4
130
131 julia> adjoint(A) == transpose(A)
132 true
133 ```
134
135 The adjoint of an `AbstractVector` is a row-vector:
136 ```jldoctest
137 julia> x = [3, 4im]
138 2-element Vector{Complex{Int64}}:
139 3 + 0im
140 0 + 4im
141
142 julia> x'
143 1×2 adjoint(::Vector{Complex{Int64}}) with eltype Complex{Int64}:
144 3+0im 0-4im
145
146 julia> x'x # compute the dot product, equivalently x' * x
147 25 + 0im
148 ```
149
150 For a matrix of matrices, the individual blocks are recursively operated on:
151 ```jldoctest
152 julia> A = reshape([x + im*x for x in 1:4], 2, 2)
153 2×2 Matrix{Complex{Int64}}:
154 1+1im 3+3im

156
157 julia> C = reshape([A, 2A, 3A, 4A], 2, 2)
158 2×2 Matrix{Matrix{Complex{Int64}}}:
159 [1+1im 3+3im; 2+2im 4+4im] [3+3im 9+9im; 6+6im 12+12im]

170 """
171 transpose(A)
172
173 Lazy transpose. Mutating the returned object should appropriately mutate `A`. Often,
174 but not always, yields `Transpose(A)`, where `Transpose` is a lazy transpose wrapper. Note
175 that this operation is recursive.
176
177 This operation is intended for linear algebra usage - for general data manipulation see
178 [`permutedims`](@ref Base.permutedims), which is non-recursive.
179
180 # Examples
181 ```jldoctest
182 julia> A = [3 2; 0 0]
183 2×2 Matrix{Int64}:
184 3 2
185 0 0
186
187 julia> B = transpose(A)
188 2×2 transpose(::Matrix{Int64}) with eltype Int64:
189 3 0
190 2 0
191
192 julia> B isa Transpose
193 true
194
195 julia> transpose(B) === A # the transpose of a transpose unwraps the parent
196 true
197
198 julia> Transpose(B) # however, the constructor always wraps its argument
199 2×2 transpose(transpose(::Matrix{Int64})) with eltype Int64:
200 3 2
201 0 0
202
203 julia> B[1,2] = 4; # modifying B will modify A automatically
204
205 julia> A
206 2×2 Matrix{Int64}:
207 3 2
208 4 0
209 ```
210
211 For complex matrices, the `adjoint` operation is equivalent to a conjugate-transpose.
212 ```jldoctest
213 julia> A = reshape([Complex(x, x) for x in 1:4], 2, 2)
214 2×2 Matrix{Complex{Int64}}:
215 1+1im 3+3im
216 2+2im 4+4im
217
218 julia> adjoint(A) == conj(transpose(A))
219 true
220 ```
221
222 The `transpose` of an `AbstractVector` is a row-vector:
223 ```jldoctest
224 julia> v = [1,2,3]
225 3-element Vector{Int64}:
226 1
227 2
228 3
229
230 julia> transpose(v) # returns a row-vector
231 1×3 transpose(::Vector{Int64}) with eltype Int64:
232 1 2 3
233
234 julia> transpose(v) * v # compute the dot product
235 14
236 ```
237
238 For a matrix of matrices, the individual blocks are recursively operated on:
239 ```jldoctest
240 julia> C = [1 3; 2 4]
241 2×2 Matrix{Int64}:
242 1 3
243 2 4
244
245 julia> D = reshape([C, 2C, 3C, 4C], 2, 2) # construct a block matrix
246 2×2 Matrix{Matrix{Int64}}:
247 [1 3; 2 4] [3 9; 6 12]
248 [2 6; 4 8] [4 12; 8 16]
249
250 julia> transpose(D) # blocks are recursively transposed
251 2×2 transpose(::Matrix{Matrix{Int64}}) with eltype Transpose{Int64, Matrix{Int64}}:
252 [1 2; 3 4] [2 4; 6 8]
253 [3 6; 9 12] [4 8; 12 16]
254 ```

420 function _dot_nonrecursive(u, v)
421 lu = length(u)
422 if lu != length(v)
423 throw(DimensionMismatch("first array has length $(lu) which does not match the length of the second, $(length(v))."))
424 end
425 if lu == 0
426 zero(eltype(u)) * zero(eltype(v))
427 else
428 sum(uu*vv for (uu, vv) in zip(u, v))
429 end
430 end
