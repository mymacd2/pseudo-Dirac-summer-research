6 """
7 r2r(A, kind [, dims])
8
9 Performs a multidimensional real-input/real-output (r2r) transform
10 of type `kind` of the array `A`, as defined in the FFTW manual.
11 `kind` specifies either a discrete cosine transform of various types
12 (`FFTW.REDFT00`, `FFTW.REDFT01`, `FFTW.REDFT10`, or
13 `FFTW.REDFT11`), a discrete sine transform of various types
14 (`FFTW.RODFT00`, `FFTW.RODFT01`, `FFTW.RODFT10`, or
15 `FFTW.RODFT11`), a real-input DFT with halfcomplex-format output
16 (`FFTW.R2HC` and its inverse `FFTW.HC2R`), or a discrete
17 Hartley transform (`FFTW.DHT`). The `kind` argument may be
18 an array or tuple in order to specify different transform types
19 along the different dimensions of `A`; `kind[end]` is used
20 for any unspecified dimensions. See the FFTW manual for precise
21 definitions of these transform types, at http://www.fftw.org/doc.
22
23 The optional `dims` argument specifies an iterable subset of
24 dimensions (e.g. an integer, range, tuple, or array) to transform
25 along. `kind[i]` is then the transform type for `dims[i]`,
26 with `kind[end]` being used for `i > length(kind)`.
27
28 See also [`plan_r2r`](@ref) to pre-plan optimized r2r transforms.
29 """
30 function r2r end

1006 # r2r inverses are normalized to 1/N, where N is a "logical" size
1007 # the transform with length n and kind k:
1008 function logical_size(n::Integer, k::Integer)
1009 k <= DHT && return n
1010 k == REDFT00 && return 2(n-1)
1011 k == RODFT00 && return 2(n+1)
1012 return 2n
