29 """
30 struct HealpixMap{T, O <: Order, AA <: AbstractArray{T, 1}} <: AbstractHealpixMap{T}
31
32 A Healpix map. The type `T` is used for the value of the pixels in a
33 map, and it can be anything (even a string!). The type `O` is used to
34 specify the ordering of the pixels, and it can either be `RingOrder`
35 or `NestedOrder`. The type `AA` is used to store the array of pixels;
36 typical types are `Vector`, `CUArray`, `SharedArray`, etc.
37
38 A `HealpixMap` type contains the following fields:
39
40 - `pixels`: array of pixels
41 - `resolution`: instance of a `Resolution` object
42
43 You can construct a map using one of the following forms:
44
45 - `HealpixMap{T, O, AA}(arr)` and `HealpixMap{T, O, AA}(nside::Number)` will use
46 `AA` as basetype
47
48 - `HealpixMap{T, O}(arr)` and `HealpixMap{T, O}(nside::Number)` will use `Array{T,
49 1}` as basetype
50
51 # Examples
52
53 The following example creates a map with `NSIDE=32` in `RING` order,
54 containing integer values starting from 1:
55
56 mymap = Healpix.HealpixMap{Int64, Healpix.RingOrder}(1:Healpix.nside2npix(32))
57
58 The call to `collect` is required to convert the range in an array.
59
60 This example creates a map in `NESTED` order, with `NSIDE=64`, filled
61 with zeroes:
62
63 mymap = Healpix.HealpixMap{Float64, Healpix.NestedOrder}(64)
64
65 Finally, the following examples show how to use `SharedArray`:
66
67 using SharedArrays
68
69 # Create a map with all pixels set to zero
70 mymap = Healpix.HealpixMap{Float64, Healpix.NestedOrder, SharedArray{Float64, 1}}(64)

72 # Create a map and initialize pixel values with a SharedArray
