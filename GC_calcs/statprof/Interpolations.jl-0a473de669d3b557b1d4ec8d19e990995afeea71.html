185 """
186 wi = WeightedIndex(indexes, weights)
187
188 Construct a weighted index `wi`, which can be thought of as a generalization of an
189 ordinary array index to the context of interpolation.
190 For an ordinary vector `a`, `a[i]` extracts the element at index `i`.
191 When interpolating, one is typically interested in a range of indexes and the output is
192 some weighted combination of array values at these indexes.
193 For example, for linear interpolation at a location `x` between integers `i` and `i+1`, we have
194
195 ret = (1-f)*a[i] + f*a[i+1]
196
197 where `f = x-i` lies between 0 and 1. This can be represented as `a[wi]`, where
198
199 wi = WeightedIndex(i:i+1, (1-f, f))
200
201 i.e.,
202
203 ret = sum(a[indexes] .* weights)
204
205 Linear interpolation thus constructs weighted indices using a 2-tuple for `weights` and
206 a length-2 `indexes` range.
207 Higher-order interpolation would involve more positions and weights (e.g., 3-tuples for
208 quadratic interpolation, 4-tuples for cubic).
209
210 In multiple dimensions, separable interpolation schemes are implemented in terms
211 of multiple weighted indices, accessing `A[wi1, wi2, ...]` where each `wi` is the
212 `WeightedIndex` along the corresponding dimension.
213
214 For value interpolation, `weights` will typically sum to 1.
215 However, for gradient and Hessian computation this will not necessarily be true.
216 For example, the gradient of one-dimensional linear interpolation can be represented as
217
218 gwi = WeightedIndex(i:i+1, (-1, 1))
219 g1 = a[gwi]
220
221 For a three-dimensional array `A`, one might compute `A/x` (the second component
222 of the gradient) as `A[wi1, gwi2, wi3]`, where `wi1` and `wi3` are "value" weights
223 and `gwi2` "gradient" weights.
224
225 `indexes` may be supplied as a range or as a tuple of the same length as `weights`.
226 The latter is applicable, e.g., for periodic boundary conditions.
227 """
228 abstract type WeightedIndex{L,W} end
